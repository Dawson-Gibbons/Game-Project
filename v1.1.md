# v1.1 Update — Leveling System Overhaul

This document describes all changes required for v1.1. Every item listed here must be implemented. If something is described as "scrapped" or "removed," all references to it in the codebase must be deleted — not commented out, not disabled, but fully removed.

---

## 1. Removals

### 1.1 — Scrap the Entire XP System

Remove the XP system completely. This includes:

- All XP variables, constants, and state (current XP, XP thresholds, XP bar, level, etc.)
- All XP gain logic (from combat, from bosses, from any other source)
- All XP-related UI elements (XP bar, XP popups, level-up notifications, level display, etc.)
- All leveling logic that was driven by XP (stat increases on level-up, unlock triggers, etc.)
- Any helper functions, utilities, or modules that exist solely to support the XP system

If a function or module has XP logic mixed in with other logic, extract and keep the non-XP parts and delete the XP parts.

### 1.2 — Scrap Boss Move Rewards

Remove the system where defeating a boss unlocks a new move. This includes:

- Any logic that grants the player a new move upon defeating a boss
- Any UI notifications or popups related to unlocking moves from boss victories
- Any checks that gate move availability behind boss defeat flags (moves are now purchased with tokens — see Section 3)

Do NOT remove the moves themselves, the move data, or the move progression order. Those are still used. Only remove the trigger that ties them to boss kills.

## 2. Bug Fixes

### 2.1 — Training Dummy Damage Bug

There is a bug where attacking the training dummy sometimes deals no damage. This needs to be fixed. Investigate and resolve whatever condition causes attacks to register as 0 or to be skipped entirely when hitting the training dummy. Every attack that lands on the dummy should always deal its correct damage value.

### 2.2 — Move Damage Adjustments

Update the following move damage values:

- **Park Hands** → now deals **15 damage**

No other move damage values are changed in this update unless they were broken by the training dummy bug fix above.

## 3. New Progression System — Tokens

Tokens are the new progression currency. They replace XP entirely.

### 3.1 — Player Starting State

When a new game begins (or after a rebirth — see Section 4), the player starts with:

- **50 Health**
- **50 Stamina**
- **0 Tokens**
- Only the original/default moveset (no unlocked moves)

### 3.2 — Earning Tokens

There are two ways to earn tokens:

**Training Dummy:**
- Defeating a training dummy always rewards exactly **1 token**.
- This is a guaranteed, consistent reward every time a dummy is defeated.

**Boss Victories (first-time kill only):**

| Boss | Token Reward |
|------|-------------|
| Alchemist A (Boss 1) | 3 tokens |
| Junky J (Boss 2) | 4 tokens |
| Dopey D (Boss 3) | 5 tokens |
| Tweaker T (Final Boss) | 50 tokens |

These boss token rewards are granted on the **first win only** per run/rebirth cycle. Defeating the same boss again does not grant tokens again (until a rebirth resets progression — see Section 4).

**Rebirth Token Multiplier:**
All token earnings (both dummy and boss rewards) are multiplied by the current rebirth multiplier. See Section 4 for multiplier values.

### 3.3 — Spending Tokens

The player can spend tokens on the following upgrades at any time (via a shop, menu, or upgrade screen):

**1-Token Upgrades (repeatable, can buy as many times as desired):**

| Upgrade | Cost | Effect |
|---------|------|--------|
| Damage Boost | 1 token | +5% damage (multiplicative, stacks) |
| Health Boost | 1 token | +5 health |
| Stamina Boost | 1 token | +5 stamina |

**5-Token Upgrade — Unlock Next Move:**

- Costs **5 tokens**.
- Unlocks the next move in the existing move progression order (the same order that was previously tied to boss defeats).
- This option should be **visible and purchasable** as long as there are still locked moves remaining.
- Once **all moves have been unlocked**, this option must **disappear entirely** from the upgrade UI. Do not show it greyed out — remove it.

**50-Token Upgrade — Rebirth:**

- Costs **50 tokens**.
- This option should **always be visible** as long as the player has not completed the final rebirth.
- See Section 4 for full rebirth details.

## 4. Rebirth System

Rebirth is a prestige mechanic. The player can rebirth up to **2 times total**. The option to rebirth should always be present in the token shop until both rebirths have been completed.

### 4.1 — First Rebirth (50 tokens)

**Cosmetic Change:**
- All `dawson_pngs` (both big and small variants) are swapped to `mary_pngs` (both big and small variants).
- This applies everywhere these images are referenced — combat sprites, menus, UI, etc.

**Progression Reset:**
- Health resets to **50**
- Stamina resets to **50**
- Moveset resets to the **original/default moveset only** (all unlocked moves are re-locked)
- All boss first-kill flags are reset (bosses can reward tokens again)
- Tokens reset to **0**
- All purchased damage boosts, health boosts, and stamina boosts are reset

**Rebirth Bonus:**
- **All token earnings are now multiplied by 2x.** This means dummy kills give 2 tokens, Alchemist A gives 6, Junky J gives 8, Dopey D gives 10, and Tweaker T gives 100.

### 4.2 — Second Rebirth (50 tokens)

**Cosmetic Change:**
- All `mary_pngs` (both big and small variants) are swapped to `ryland_pngs` (both big and small variants).

**Progression Reset:**
- Identical to first rebirth: health to 50, stamina to 50, moveset to default, boss flags reset, tokens to 0, all purchased boosts reset.

**Rebirth Bonus:**
- **All token earnings are now multiplied by 3x.** This means dummy kills give 3 tokens, Alchemist A gives 9, Junky J gives 12, Dopey D gives 15, and Tweaker T gives 150.

### 4.3 — After Final Rebirth

Once the second rebirth has been completed, the rebirth option should be **removed from the token shop**. The player continues playing on the 3x multiplier with ryland_pngs permanently.

## 5. Token Shop — UI & Placement

### 5.1 — Location

The token shop must be displayed in the **bottom-right corner** of the map screen. It should **always be visible** whenever the player is on the map screen — it is not hidden behind a button or toggle. It is a persistent, always-on-screen UI element on the map.

### 5.2 — Display

The shop should show the player's **current token count** prominently so the player always knows what they can afford.

Each purchasable option should display its **name, cost, and effect** clearly. Options the player cannot currently afford (not enough tokens) should appear **greyed out or visually dimmed** but still visible, so the player knows what they are saving toward.

### 5.3 — Shop Contents

The shop contains the following options, rendered in this order:

1. **+5% Damage** — 1 token (always visible, repeatable)
2. **+5 Health** — 1 token (always visible, repeatable)
3. **+5 Stamina** — 1 token (always visible, repeatable)
4. **Unlock Next Move** — 5 tokens (visible only while locked moves remain; disappears entirely once all moves are unlocked)
5. **Rebirth** — 50 tokens (visible only while rebirths remain; disappears entirely after the second rebirth is completed)

### 5.4 — Interaction

When the player clicks/selects a shop option and has enough tokens:

- Deduct the token cost immediately.
- Apply the effect immediately (stat boost takes effect right away, move is usable right away, rebirth triggers right away).
- Update the UI to reflect the new token balance and any changes (e.g., the move unlock option disappearing).

When the player clicks an option they cannot afford, nothing happens — no purchase, no error popup, just a no-op. The greyed-out visual state should make it clear they can't buy it yet.

### 5.5 — Persistence

The shop must respect the current game state at all times. If the player leaves the map screen (enters combat, etc.) and returns, the shop should reflect the current token count and available options accurately. The shop does not have its own separate state — it reads directly from the player's token balance, rebirth count, and move unlock progress.
